    # Installation

Tailwind CSS can be installed in multiple ways depending on the tech stack.  
This guide breaks down installation via:

- CDN (quick prototyping)
- npm (recommended)
- Framework setups (Next.js, Vite, React, Vue, Angular)

---

# üöÄ 1. Install Tailwind via CDN (Fastest for Prototyping)

Use this method **only** for quick experiments or CodePen-style demos.

```html
<script src="https://cdn.tailwindcss.com"></script>
```

    Optional: Inline config

    <script>
  tailwind.config = {
    theme: {
      extend: {
        colors: {
          primary: '#4f46e5',
        },
      },
    },
  };
</script>

    ‚ö†Ô∏è Not recommended for production. No PurgeCSS ‚Üí huge CSS size. No custom configuration. No JIT mode.

    üì¶ 2. Install using npm (Recommended)

    Step 1 ‚Äî Create project

    mkdir my-project
cd my-project
npm init -y

    Step 2 ‚Äî Install Tailwind + PostCSS + Autoprefixer

    npm install tailwindcss postcss autoprefixer

    Step 3 ‚Äî Create config files

    npx tailwindcss init -p

    This generates:

    tailwind.config.js
postcss.config.js

    üìå 3. Add Tailwind to your CSS

    Create src/input.css:

    @tailwind base;
@tailwind components;
@tailwind utilities;

    üîß 4. Configure Template Paths (Important)

    In tailwind.config.js:

    module.exports = {
  content: ['./src/**/*.{html,js}'],
  theme: {
    extend: {},
  },
  plugins: [],
};

    ‚ö†Ô∏è Catch Point: If content paths are wrong, Tailwind classes will not apply or will be missing after build.

    üõ† 5. Build CSS (Development Mode)

    npx tailwindcss -i ./src/input.css -o ./dist/output.css --watch

    Include in HTML:

    <link rel="stylesheet" href="./dist/output.css" />

    ‚ö° 6. Installing Tailwind in Popular Frameworks

    üü¶ Next.js

    npx create-next-app@latest my-app
cd my-app
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init -p

    Content paths:

    content: ['./pages/**/*.{js,ts,jsx,tsx}', './components/**/*.{js,ts,jsx,tsx}'];

    Add Tailwind to globals.css.

    üü® Vite (React, Vue, Svelte)

    npm create vite@latest my-app
cd my-app
npm install
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init -p

    üü¶ React (CRA)

    npx create-react-app my-app
cd my-app
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init -p

    Content paths:

    content: ['./src/**/*.{js,jsx,ts,tsx}'];

    üüß Angular

    ng new my-app
cd my-app
ng add @ngneat/tailwind

    üéØ Summary

    • CDN = quick prototype
    • npm = full power + customization
    • Frameworks require template paths
    • tailwind.config.js is core for extending themes

# Tailwind Configuration (tailwind.config.js)

The `tailwind.config.js` file controls theme customization, plugins, variants, dark mode, and paths.
Understanding this file is essential for advanced Tailwind usage.

---

# üèó Basic Structure

```js
module.exports = {
  content: ["./src/**/*.{html,js}"],

  theme: {
    extend: {},
  },

  plugins: [],
};

    üîπ content

    Defines files where Tailwind should scan for class names.

    content: ['./index.html', './src/**/*.{html,js,jsx,ts,tsx,vue}'];

    ‚ö†Ô∏è If you forget to include a directory, Tailwind will purge required styles.

    üîπ theme.extend

    Use extend, not override.

    theme: {
  extend: {
    colors: {
      primary: "#4f46e5",
      brand: {
        light: "#c7d2fe",
        DEFAULT: "#6366f1",
        dark: "#4338ca"
      }
    },
    spacing: {
      128: "32rem",
      144: "36rem"
    },
  },
}

    üé® Custom Fonts

    extend: {
  fontFamily: {
    inter: ["Inter", "sans-serif"],
  },
}

    Usage:

    <p class="font-inter">Hello</p>

    üåë Dark Mode

    darkMode: 'class';

    Usage:

    <html class="dark"></html>

    Or toggle via JS.

    üì¶ Plugins

    plugins: [require('@tailwindcss/forms'), require('@tailwindcss/typography')];

    üß© Default Theme Reference

    You can view the full Tailwind theme:

    npx tailwindcss theme

    üéØ Best Practices

    • Always use extend instead of replacing Tailwind defaults.
    • Store colors and spacing in variables for consistency.
    • Avoid over-expanding your config file.
    • Group related config in objects for readability.

# PostCSS, PurgeCSS, and Tailwind JIT Explained

Tailwind relies on PostCSS for processing, and uses a purge-like mechanism to remove unused CSS.

---

# üß© What is PostCSS?

A CSS processor that handles tasks like:

- autoprefixing
- Tailwind processing
- minification

Your `postcss.config.js` will look like:

```js
module.exports = {
  plugins: {
    tailwindcss: {},
    autoprefixer: {},
  },
};

    üßπ Purging Unused CSS

    Before Tailwind v3, PurgeCSS was required. Now Tailwind uses content scanning + JIT engine, automatically removing unused classes.

    ‚ö†Ô∏è If classes do not appear in content paths ‚Üí they are removed.

    ‚ú® JIT Mode (Now Always Enabled)

    JIT compiles CSS on-demand and offers:

    • instant builds
    • arbitrary values (mt-[23px])
    • tiny production output

    üß± Safelisting (Must-Know)

    Sometimes you generate dynamic class names in React/Vue, e.g.:

    <div className={`bg-${status}-500`}></div>

    Tailwind cannot detect these dynamic class names.

    Solution:

    safelist: [
  { pattern: /bg-(red|green|blue)-(500|600)/ }
],

    üß™ Example: Production Build

    NODE_ENV=production npx tailwindcss -o dist/output.css --minify

    ‚ö†Ô∏è Common Pitfalls

    • Class names constructed as strings will be purged.
    • Missing paths ‚Üí utilities disappear.
    • Do not integrate old PurgeCSS plugins; Tailwind already handles purging.

# Setup Pitfalls

Tailwind is powerful ‚Äî but beginners often face setup issues.
Here are the most common ones and how to fix them.

---

# ‚ùå 1. Wrong Content Paths

Problem:

```js
content: ["./src/*.html"]

    This excludes nested folders ‚Üí Tailwind purges everything.

    Correct:

    content: ['./src/**/*.{html,js,jsx,ts,tsx,vue}'];

    ‚ùå 2. CSS File Not Included

    Make sure this file exists and is imported:

    @tailwind base;
@tailwind components;
@tailwind utilities;

    Common mistake:

    <link href="input.css" />

    Should be:

    <link rel="stylesheet" href="./dist/output.css" />

    ‚ùå 3. Missing PostCSS Plugins

    postcss.config.js must include tailwindcss and autoprefixer.

    ‚ùå 4. Using Tailwind CDN in Production

    This produces:

    • Large bundle sizes
    • No purging
    • No custom config

    ‚ùå 5. Dynamic Class Names Not Detected

    Example:

    <div className={`bg-${color}-500`}></div>

    Fix:

    safelist: [/bg-(red|blue|green)-(500|600)/];

    ‚ùå 6. Tailwind Not Rebuilding (Watch Not Working)

    Caused by:

    • Wrong paths
    • Wrong -i and -o
    • File inside ignored directory

    ‚ùå 7. Overusing @apply

    Bad:

    .btn {
  @apply px-4 py-2 bg-blue-500 text-white font-bold rounded shadow;
}

    ‚ö†Ô∏è This recreates Bootstrap-style CSS ‚Üí defeats utility-first purpose.

    Only apply layout-level patterns.

    üéØ Summary

    • Correct paths are essential
    • Avoid dynamic string class names
    • Do not abuse @apply
    • Ensure proper PostCSS setup
