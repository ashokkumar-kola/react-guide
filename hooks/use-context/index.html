    useContext

    1. üéØ Purpose

    Global State Sharing ‚Äî enables data sharing across components without prop drilling.

    üß† Key Uses

    • Avoids deeply nested prop passing
    • Centralizes global or shared state
    • Enables cross-component communication

    üí° Ideal for themes, authentication, language, and app-wide state.

    2. üìù Syntax

    Create Context

    const MyContext = React.createContext(defaultValue);

    Provide Context

    <MyContext.Provider value={currentValue}>
  {/* Child components */}
</MyContext.Provider>

    Consume Context

    const value = useContext(MyContext);

    ‚öôÔ∏è useContext() allows any descendant component to access context value directly.

    3. üîß Basic Pattern

    1Ô∏è‚É£ Create Context

    2Ô∏è‚É£ Provide Value

  const [theme, setTheme] = useState('light');

  return (

      <Header />

    3Ô∏è‚É£ Consume Value

  const { theme, setTheme } = useContext(ThemeContext);

    </header>
  );
}

    5. üèóÔ∏è Advanced Patterns

    üß± Multiple Contexts

    function App() {
  return (
    <ThemeContext.Provider value={theme}>
      <AuthContext.Provider value={auth}>
        <UserContext.Provider value={user}>
          <Component />
        </UserContext.Provider>
      </AuthContext.Provider>
    </ThemeContext.Provider>
  );
}

    üí° Combine multiple contexts when different data domains are needed.

    ‚öôÔ∏è Custom Provider Component

    function ThemeProvider({ children }) {
  const [theme, setTheme] = useState('light');

  const value = {
    theme,
    toggleTheme: () =>
      setTheme((prev) => (prev === 'light' ? 'dark' : 'light')),
  };

  return (
    <ThemeContext.Provider value={value}>{children}</ThemeContext.Provider>
  );
}

    üß≠ Default Values

// Used if no Provider exists in component tree

    6. ‚ö†Ô∏è Performance Considerations

    üß† Re-render Optimization

    // ‚ùå Causes re-renders: new object every render

// ‚úÖ Memoized value object

<ThemeContext.Provider value={themeValue}>

    üß© Separate Contexts for Different Concerns

    // ‚ùå One giant context

// ‚úÖ Split logically

const UserContext = React.createContext();

    ‚úÖ Smaller, focused contexts = fewer re-renders and better scalability.

    7. üö® Common Mistakes

    ‚ùó Missing Provider

    8. üí° Best Practices

    ‚úÖ Do‚Äôs

    1. Create contexts close to where they‚Äôre used
    2. Use meaningful names (e.g., UserContext, not Context1)
    3. Provide default values for better testing and safety
    4. Split logically ‚Äî use multiple small contexts
    5. Memoize context values to prevent extra re-renders

    ‚ùå Don‚Äôts

    • Avoid a ‚Äúmega context‚Äù for everything
    • Don‚Äôt skip wrapping consumers with providers
    • Don‚Äôt create new value objects inline each render

    9. üß© Custom Hook Pattern

    Encapsulate context logic inside a custom hook for reusability and safety.

    function useTheme() {
  const context = useContext(ThemeContext);
  if (!context) {
    throw new Error('useTheme must be used within a ThemeProvider');
  }
  return context;
}

    üí° Simplifies imports and ensures context is properly used.

    10. üéØ Key Takeaways

    • üß≠ Solves prop drilling ‚Äî share state globally
    • ‚öôÔ∏è Provider / Consumer pattern ‚Äî wrap and consume
    • üöÄ Optimize performance ‚Äî memoize context values
    • üß© Use multiple contexts ‚Äî avoid overloading one
    • üîß Provide defaults ‚Äî prevent runtime errors

    ‚úÖ Summary: useContext is the go-to React Hook for sharing global or cross-component state efficiently. When used with memoization, custom hooks, and context separation, it leads to clean, scalable, and high-performance React architectures.
